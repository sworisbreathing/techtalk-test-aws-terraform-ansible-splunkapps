Cluster master view

This view queries the master node of a Splunk Enterprise cluster to provide an overview of the cluster.

This view shows results only if you access it from a cluster master. It does not display data if you access the view from any other instance in a cluster.

If you are running Splunk Enterprise 6, consult the updated version of this view.

For an overview of Splunk Enterprise clusters and index replication, refer to About clusters and index replication in the Splunk Enterprise documentation.

Cluster replication info

Provides an overview of your cluster configuration.

Bucket count

Provides a visualization of the relative number of bucket copies, by index or peer, in a cluster. The bucket count includes the primary bucket and all copies.

Mouse over each area of the chart to view the actual number of replicated buckets present in each index or peer.

Peer bucket count by status

View the number of bucket copies for each peer, based on the bucket replication (complete or in-flight) or searchable (searchable or unsearchable) status.

Use the logarithmic scale when you encounter large counts from a single peer.

Bucket information

View details about each bucket in your cluster, listed by index. Note that each result in the table shows a unique *bucket ID*, which will exist as several copies on the cluster peers. Peer bucket status column shows peers holding copies of a specific bucket, indicating those that hold primary, searchable, or unsearchable copies.

Searches used in this view

Click the View results link in each panel to open the search in the Search view. You can inspect, modify, and export the search results from the Search view.
