Trusted digital certificates installed on SplunkWeb

Default self-signed certificates may lead to MITM (man in the middle) attacks because all default Splunk installations use self-signed default digital certificates. This makes it easy to spoof a Splunk server.

More

Splunk ships with, and is configured to use, a set of default certificates. These certificates will discourage casual snoopers but could still leave you vulnerable because the root certificate that ships with Splunk is the same root certificate in every download, and anyone with the same root certificate can authenticate. Read more here

Splunk running as a non-root user

Use a non-root user for running Splunk with a null shell to keep Splunk from root level exploits.

Default Admin Credentials Disabled

Make sure you use a different password or passphrase from the default credentials. That will reduce successful attacks from automated scanners with default passwords in them.

More

When you first installa and run Splunk it will come with the default credentials username: admin and password: changeme. It is very important to change these credentials as they are widely known and easily guessable.
