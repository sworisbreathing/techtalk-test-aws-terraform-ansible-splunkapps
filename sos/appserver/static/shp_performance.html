Search Head Pooling Performance

Use this view to analyze the usage and performance of the NFS shared storage that hosts the files and directories shared by all members of a search head pool. The data presented highlights the relationship between usage and performance both at the storage (NFS) and application (Splunk Enterprise) levels.

About this view

The following requirements need to be fulfilled for this view to present information:

  • Search head pooling is configured in your deployment.

  • The nfs-iostat_sos.py scripted input is enabled on each search head of the pool. This data input collects NFS metrics for usage and performance of the shared storage device used by the search head pool.

  • The view is loaded on a search head member of the pool or all search heads in the pool forward the events they collect to the indexers (this is the recommended approach).

Panels in this view

Pooling Shared Storage – I/O Throughput vs. Performance

This panel compares usage vs. performance at the storage level by overlaying two key metrics captured by the nfs-iostat_sos.py scripted input:

  • IOps – I/O operations per second that splunkd imposes on the shared storage device.

  • RTT – Round trip time, in milliseconds, for three particular I/O operations: access(), getattr() and lookup(). These types of I/O operations represent the bulk of the workload that splunkd places on the shared storage device.

Splunk UI and Search – Usage vs. Responsiveness

This panel compares usage vs. performance at the application level by overlaying some key metrics captured in Splunk Enterprise internal logs:

  • Search dispatch time – The time (in seconds) it takes for splunkd to service an ad hoc search request initiated by a user from Splunk Web.

  • View service time – The time (in seconds) for splunkd to serve a view in Splunk Web following a user request.

  • Search concurrency – The number of searches running concurrently at a specific time.

Top 10 IOps Peaks by Host

This panel displays the top ten busiest moments in terms of total IOps serviced by the shared storage device, split by host in the search head pool.

IOps per Operation Category

This panel measures the IOps imposed by splunkd on the shared storage device, grouped by major operation category.

Searches used in this view

Click the View results link in each panel to open the search in the Search timeline. You can inspect, modify, and export the search results from the Search timeline.
