Advanced XML: How modules work

As mentioned in the Advanced XML: Hello World, when a module contains another module, the contained module (child module) inherits whatever changes are affected by the first module (the parent module). There are several kinds of modules, but you can think of them as falling into two major types:

Modules that build a search

Modules that don't build a search

Modules that build a search

You can tie together several modules to build a search that is kicked off as a job. To build a search, you need a base search string from a search module. For example:

SearchBar

HiddenSearch

HiddenSavedSearch

other search module

Then, you can add an optional start time argument and an optional end time argument.

Then add intentions. Intentions are layers added to the base search, typically by interactive elements in the interface, that affect the eventual search that is run.

Intentions

Most views contain a base search string – if you don't explicitly set one, it is implicitly "*".

SearchBar, HiddenSearch, and HiddenSavedSearch are the most common modules that specify the base search.

On top of that base search, other modules can layer intentions such as addterm, stringreplace, plot, negateterm, addtermgt and others.

Each of those intentions can be added on top of any base search string, affecting the search that gets run.

Eventually, the base search and all the intentions layered on top of it reach a module that requires that kicks off the search (for example, JobStatus, EventsViewer, FlashTimeline, and others). The base search plus the intentions becomes flattened into the search that runs.

Note: The stringreplace intention is a bit of an outlier because it requires the base search string to contain embedded tokens ($foo$, for example).

Modules that don't build a search

This covers a huge set of diverse modules – everything from setting the type of chart (for example, pie chart, line chart, and others) to setting the number of results per page, to changing sort order, to specific information about objects the user clicked since the view loaded.
