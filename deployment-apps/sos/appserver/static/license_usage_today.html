License Usage - Today

About this view

These panels shows license usage and warnings for the current day. The licenser's day ends at midnight in the time zone of the License Master.

Load this view on your License Master instance

This view queries REST API endpoints to gather information about your license usage. As such, this view should only be loaded on your Splunk License Master instance.

Version compatibility

  • Splunk Enterprise 5.x
    This view is supported on these versions.

  • Splunk Enterprise 6.x
    This view has been disabled for Splunk Enterprise 6.0 and later versions. Splunk Enterprise versions 6.x ship with a built-in License Usage Report View.

Troubleshooting this view

Common reasons for data not to show in this view:

  • The view is not being consulted on the License Master instance.

  • The license_usage.log file created on the License Master is not being indexed.

  • The license_usage.log file is being indexed but its events are being forwarded to instances which are not reachable by distributed search from the License Master.

Panels in this view

Todays License Usage

Shows the total license usage for the current day as well as the total daily license quota across all pools.

Todays License Usage per Pool

Shows the license usage for the current day and the daily license quota for each configured pool.

Today's Percentage of Daily License Quota Used per Pool

Shows the percentage of the daily license quota that has been consumed for each configured pool.

Pool Warning Information

Shows license warnings, both soft and hard, that each pool has received in the past 30 days. Read About license violations in the Splunk Enterprise Admin manual to learn more about warnings and violations.

Slave Warning Information

For each license slave, shows:

  • GUID

  • Pool membership

  • Number of hard warnings

  • Violation status

Searches used in this view

Click the View results link in each panel to open the search in the Search timeline. You can inspect, modify, and export the search results from the Search timeline.
