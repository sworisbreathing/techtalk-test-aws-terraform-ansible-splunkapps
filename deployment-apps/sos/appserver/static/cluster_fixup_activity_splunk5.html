Bucket Fix-up Activity

This view shows results only if you access it from a cluster master node. It does not display data if you access the view from any other instance in a cluster.

If you are running Splunk Enterprise 6, consult the updated version of this view.

Count of buckets to fix, by reason

Provides a count of buckets that are in need of fixing, grouped by fix reason.

List of buckets to fix, by reason

Displays a list of buckets that are in need of fixing, grouped by fix reason.

List of buckets to fix

Displays a full, searchable list of buckets that are in need of fixing.

Recent bucket-fixing activity

Displays events recording bucket fix ups, as recorded on the master. Check the "Show fsck events on peers" checkbox to also see events recording fsck activity on the peers.

Generation-blocking bucket fixes

Displays two tables:

  • The first table indicates whether the cluster can publish a new generation.

  • The second table provides a list of buckets with less than one searchable copy. Buckets with less than one searchable copy are the primary blockers for the publication of a new generation.

Searches used in this view

Click the View results link in each panel to open the search in the Search view. You can inspect, modify, and export the search results from the Search view.
