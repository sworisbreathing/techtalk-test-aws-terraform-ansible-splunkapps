Switcher modules

Switchers are a way to create tabs or pulldown elements that users can use to switch the content or UI controls displayed. They may seem similar to lister modules, but they are qualitatively different. Switchers create a fork between arbitrarily different config branches underneath. The choice doesn't influence any individual search in the child branches. Listers, on the other hand, allow the user to insert input that affects the searches underneath – they do not switch one set of modules out and replace it with another set of modules.

Switcher classes available are:

  • PulldownSwitcher

  • TabSwitcher

  • LinkSwitcher

  • ButtonSwitcher

Note: ButtonSwitcher is a little misnamed (it actually displays square icons) and is a bit complex to implement. You need to create custom images for ButtonSwitcher and it usage within the Splunk timeline is a special case.

The example in this view uses a LinkSwitcher with 4 children.

The first child uses a HiddenSavedSearch plus HiddenChartFormatter plus JobProgressIndicator plus JSChart pattern

The next two use two different HiddenSearch plus HiddenChartFormatter plus JobProgressIndicator plus JSChart patterns
