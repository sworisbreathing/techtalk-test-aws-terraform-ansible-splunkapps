Advanced XML: Layout

These are the main considerations when specifying a layout for a view:

  • Template for the view

  • Chrome

  • Layout panel

Template for the view

Views in Splunk use Mako templates to define the page layout for a view. Typically, you select either the dashboard or search attribute to the <view> tag to specify the template. Most of the advanced XML examples in this app specify the dashboard template. However, there are a series of examples illustrating the search template.

Chrome

Chrome represents the layout panels that define the appearance of a dashboard. You can choose from the following layout panels:

  • messaging
    Displays messages to a user.

  • appHeader
    Displays the bar at the top of most views.

  • navigationHeader
    Use with the AppBar module to display navigation for an app.

  • viewHeader
    A header panel typically used to display a title using the TitleBar module.

layoutPanel and other module attributes

The layoutPanel attribute can specify the position of modules in a view. The search and dashboard templates allow for different layoutPanel attributes. For a dashboard, use attributes to the layoutPanel to specify the row and column order of modules the dashboard. For the default dashboard template, you can specify any number of rows, but only up to three columns.

<module name="ModuleName" layoutPanel="panel_rowN_colN">

Use the group attribute to provide a header for a panel, and autoRun to run the search in the panel when the panel loads.

<module name="ModuleName" layoutPanel="panel_row1_col1" group="My Panel Header" autoRun="True">

Any module can have a layoutPanel attribute set, meaning that the module renders into the specified layoutPanel. Child module's inherit the layoutPanel attribute unless they specify their own layoutPanel attribute.
