Two listers driving multiple elements

This example shows two listers (one static and one dynamic) plus a TimeRangePicker that drives multiple elements.

Search listers take parameters that allow them to incorporate intentions and time ranges from interactive elements upstream. In this way the value pulldown runs its internal search whenever you change the selected field.

This example illustrates some of the complexity of nesting modules.
