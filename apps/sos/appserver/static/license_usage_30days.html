License Usage - Last 30 Days

About this view

The panels in this view show license usage for the current licenser time window, which spans back thirty days.

Load this view on your License Master instance

This view leverages information about license usage that is usually only available on the License Master instance. As such, this view should only be loaded on that instance.

Search controls

  • Pool dropdown
    Select All Pools or a specific pool.

  • Split by dropdown
    Select the field to break down license usage against.

    • no split, indexer, pool
      Populates the panel with data from license_usage.log. If your License Master is down at its local midnight, it does not generate a RolloverSummary event for that day – that day's data does not appear in the panels.

    • source, source type, host
      Populates the panel by aggregating the usage statistics reported by the slaves every minute into one-day data points. Splunk recommends accelerating the search that powers these panels.

Version compatibility

  • Splunk Enterprise 5.x
    This view is supported on these versions.

  • Splunk Enterprise 6.x
    This view has been disabled for Splunk Enterprise 6.0 and later. These versions ship with a built-in License Usage Report View.

Troubleshooting this view

Common reasons for data not to show in this view:

  • The view is not being consulted on the License Master instance.

  • The license_usage.log file created on the License Master is not being indexed.

  • The license_usage.log file is being indexed but its events are being forwarded to instances which are not reachable by distributed search from the License Master.

  • If your license master instance is down at its local midnight, it does not generate a RolloverSummary event for that day, and you do not see that day's data in these panels.

Accelerating the searches for this view

Acceleration for the searches powering this view is disabled by default. Acceleration of searches apply to the following split-by selections, which are powered by one datacube-generating search named sos_license_usage_datacube:

  • host

  • source

  • source type

Enable acceleration for the "sos_license_usage_datacube" search, but only on the License Master instance. Note that depending on the amount of data already available, it may take a significant amount of time for the summary to be fully built.

Squashing

Periodically the indexer reports to the license manager the data indexed, broken down by source, sourcetype, host, and index. If the number of distinct (source, source type, host, index) tuples in these reports grows over the squash_threshold, Splunk Enterprise squashes the host and source values and only reports a breakdown by sourcetype and pool.

The squashing threshold is configurable in server.conf. You can very carefully increase the squash_threshold in the [license] stanza of the license slaves. Note that the sourcetype field is not subject to squashing.
