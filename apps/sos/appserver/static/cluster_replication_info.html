Index replication information

This view provides several panels for monitoring the activity of peer nodes in a Splunk Enterprise cluster. Information presented includes peer status, index replication errors, and data throughput with respect to index replication. To some extent, index replication performance is measured by examining the fill ratio of the outgoing replication queue.

For an overview of Splunk Enterprise clusters and index replication, refer to About clusters and index replication in the Splunk Enterprise documentation.

Cluster peer status

Provides an overview of peer nodes, listing which peers are registered and their status.

Index replication errors – last 24 hours.

Lists cluster-specific errors, including bucket replications errors, starting with the most recent error. By default, similar errors are grouped into one entry. Deselect Group similar events to view all errors.

To view errors prior to the last 24 hours, click View results and modify the search time window.

Outgoing network replication throughput | Incoming network replication throughput

These two panels provide measurements (average and maximum) and trend of bucket replication throughput between peers.

Fill ratio of outgoing replication queue by peer

Use this panel to check the outgoing queue of source peers sending replicated copies to target peers. This queue holds events to be replicated and is shared by as many buckets as there are that are being replicated. Events in the queue can have multiple destinations.

The queue can become backed up if one or more target peers are slow in receiving data. In this case, check the Distributed Indexing Performance view to look for a possible indexing bottleneck.

Function specifies how to measure the Fill ratio.

Bucket Span specifies the time span between data points on the Time axis.

Searches used in this view

Click the View results link in each panel to open the search in the Search view. You can inspect, modify, and export the search results from the Search view.
