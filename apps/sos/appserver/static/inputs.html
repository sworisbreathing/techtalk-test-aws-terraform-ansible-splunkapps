Data inputs overview

Use this view to review and analyze the data inputs of the selected Splunk Enterprise instance.

Populate each of the Data input details panels on the right by selecting a data input from the corresponding Data inputs panel on the left.
Clicking an item listed in a Data inputs panel provides additional information in the corresponding Data input details panel.

A list of input-related errors found in splunkd.log is also available.

S.o.S gathers information about the data inputs enabled on the selected instance by accessing the /services/admin/meta/enabled_inputs REST API endpoint.

To populate the Recent input error messages panel, S.o.S searches the _internal index for splunkd.log events that have been recorded with a log level of ERROR by data input components.

Enabled data inputs

The pie chart shows the type and number of data inputs configured and enabled on the selected Splunk instance.

  • Mouse over slices in the chart to view more information about the inputs.

  • Select a slice to list the inputs of that type.

  • Select an input from the list to display the stanza defining that input plus other details, if available.

Search enabled data inputs

Provides a search bar to search for specific enabled data inputs using keywords.

From the list of found enabled inputs, select an input to display details.

Recent input error messages

Lists the most recent error messages found in splunkd.log and recorded by data input components such as the tailing processor.

Use the Cluster events checkbox to aggregate similar error messages together.

Searches used in this view

Click the View results link in each panel to open the search in the Search view. You can inspect, modify, and export the search results from the Search view.
