Metrics

This view displays data from the metrics.log log file, available in $SPLUNK_HOME/var/log/splunk/.

The metrics log file (metrics.log) contains a sampling of the top ten items for each indexing throughput category (index, source, sourcetype, and hosts) for 30 second intervals, based on the amount of event rawdata indexed.

Important: Events from the metrics.log file per_*_thruput groups do not provide an exact accounting of all your indexed data. It just shows the top 10 series recorded for each metadata field for each 30 second window.
For example, if you have 2000 active hosts, you cannot expect to see all of them accounted for in events from metrics.log / group=per_host_thruput.

For more information on internal metrics and log files, refer to:

  • What Splunk Enterprise logs about itself from the Splunk Enterprise Admin manual.

  • Work with metrics.log from the Splunk Enterprise Admin manual.

  • Metrics.log information from the Splunk Community Wiki.
