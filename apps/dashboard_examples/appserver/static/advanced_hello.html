"Hello, Advanced XML"

Data flow, or intentions, are determined by containment in advanced XML. If you have two modules in a view, and neither module 'contains' the other in the XML, the modules really know nothing about each other and they function completely independently. If, on the other hand you have a module nested inside another module (for example, a module nested inside the HiddenSearch module) the nested module inherits the effects and settings of the parent module.

This view provides an example of a HiddenSearch module that contains a SimpleResultsTable module. Consider this example to be the advanced XML equivalent to "Hello World." The SimpleResultsTable has access to the search results from its parent module, the HiddenSearch module.

Here is an example of the base code for a HiddenSearch module:

<module name="HiddenSearch" layoutPanel="panel_row2_col1"
    group="Hello World - how to wire two modules together" autoRun="True">
  <param name="search">
    index=_internal source=*metrics.log group=per_sourcetype_thruput
    | chart sum(kb) by series | sort -sum(kb) | head 10
  </param>
  <param name="earliest">-1h</param>
  <module name="SimpleResultsTable"></module>
</module>
